FEATURE 1 — SETTINGS BUTTON ON DASHBOARD

The dashboard currently lacks a Settings menu.

Create a Settings button in the top-right corner of the dashboard.

When clicked, the button should open a dropdown menu.

The dropdown must include the following options:

Change Date Format

Allow the user to choose between:

DD/MM/YYYY

MM/DD/YYYY

YYYY-MM-DD

Change Background Color / Theme

Light Mode

Dark Mode

Custom background color picker

Currency Settings

Allow user to select currency symbol

Examples: $, ₦, €, £

Notification Settings

Toggle Low Stock Notifications ON/OFF

Data Export

Export inventory to CSV or Excel

Reset Settings

Technical Requirements:

• Settings must persist using local storage or database
• Settings must apply immediately after selection
• Theme change must update the entire UI

FEATURE 2 — FIX MANAGE BUTTON NAVIGATION

Currently, clicking the Manage button does not lead to the correct inventory item.

Fix the navigation so that:

When the user clicks Manage on a specific inventory item, the system must navigate to that item's detail page.

Correct behavior:

Dashboard → Manage button → Item Detail Page

Implementation Requirements:

• Each item must have a unique ID
• The Manage button must route to:

/inventory/{itemId}

Example:

/inventory/48392

Ensure:

• Correct item data loads dynamically
• No page reload errors
• Proper error handling if item ID does not exist
